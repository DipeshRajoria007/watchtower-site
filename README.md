# Watchtower Site

This repo is the front porch for Watchtower.

Not the engine room.
Not the sidecar.
Not the tray icon.

The front porch.

It exists to make one thing obvious very quickly:

Slack is where engineering requests show up. Watchtower is how those requests stop turning into theater, guesswork, and "who owns this?" energy.

## What This Page Is Selling

Watchtower is a macOS-native product for teams that want autonomous developer workflows without surrendering visibility.

The site tells that story in a more cinematic way than the app repo does:

- Slack mention in
- the right workflow wakes up
- action happens
- traces, proof, and learning stay visible

The whole page is aimed at engineering leaders who like automation, but not enough to trust it blindly.

## What You Will See

- An editorial command-room visual style instead of generic SaaS gradients
- Conceptual product theater inspired by the real app: Slack intake, execution traces, autopilot, and learning loops
- A single-page flow built to feel more like a controlled operating system than a brochure
- A straight path to book a demo and a second path to inspect the actual product repo

## Why The README Is Not Trying To Be Cleverly Technical

Because this repository is supposed to feel like a pitch you can walk through, not a setup wizard that immediately asks you to become build tooling.

If you are here, you probably want one of two things:

1. See how Watchtower is being framed as a product.
2. Run the page locally and tweak the visuals.

Both are valid. Only the second one needs commands.

## Run It Locally

```bash
npm install
npm run dev
```

If you want the production build:

```bash
npm run build
```
